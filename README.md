#  WebsiteRestaurant

🌿 Ресторан «Берлога» — Система управления и бронирования

Это современное веб-приложение для автоматизации работы ресторана: от управления афишей мероприятий до гибкой системы бронирования столиков с уведомлениями.

🚀 Основные возможности
•  Система бронирования: Удобный выбор столиков для авторизованных и анонимных пользователей.
•  Управление мероприятиями: Полноценный календарь событий (анонсы, подробности, архив) с кэшированием для быстрой загрузки.
•  Личный кабинет клиента: История бронирований, отслеживание статусов в реальном времени, возможность оценки посещения и написания отзывов.
•  Умная модерация: Разграничение прав доступа (Staff/Moderator) для управления контентом и подтверждения броней.
•  Email-уведомления: Автоматические приглашения на новые мероприятия и напоминания о предстоящей брони за 1 час до начала (через Celery).
•  Обратная связь: Система оценок и комментариев после завершения визита для улучшения сервиса

 🛠 Технологический стек
•  Backend: Python 3.13, Django 5.1+ (Class-based views)
•  База данных: PostgreSQL
•  Фоновые задачи: Celery + Redis
•  Кэширование: Redis (Django Cache Framework)
•  Стиль кода: Flake8, Isort
•  Тестирование: Django Test, Coverage.py
•  Управление зависимостями: Poetry / Pip
•  Деплой: Docker, Docker Compose, GitHub Actions (CI/CD)

 📋 Требования

•  Python 3.13 или выше
•  PostgreSQL 15+
•  Redis сервер (запущенный локально или в Docker)

 ⚙️ Установка и настройка

1. Клонируйте репозиторий
2. Установите зависимости
3. Настройте переменные окружения:
  Создайте файл .env в корне проекта или пропишите в settings.py
    SECRET_KEY=django_secret_key
    DEBUG=True
    CELERY_BROKER_URL=redis://localhost:6379/0
4. Примените миграции
5. Создайте суперпользователя (для доступа к админке)

🏃 Шаги для запуска проекта через
docker-compose:
1. Копирование Dockerfile и Docker-compose.yml
2. Настройка файла .env (Конфиденциальность)
3. Проверка файлов исключений .gitignore и .dockerignore
Первый запуск
1. docker compose down -v # Остановит и удалит все контейнеры, сети, и ТОМЫ (postgres_data, media)
2. docker compose up -d --build # Запустит все сервисы и соберет образы
*  -d: Запустить в фоновом режиме.
  •  --build: Пересобрать образы (особенно важно после изменений в Dockerfile).
3. docker compose ps # Проверит статус контейнеров(Все сервисы должны быть в статусе Up или Running)
4. docker compose logs -f  # Покажет логи (для отладки)
5. docker compose exec app poetry run python manage.py migrate # Выполнит миграции Django
6. docker compose exec app poetry run python manage.py createsuperuser # Создаст суперпользователя Django
7. Проверьте веб-приложение:
  Откройте браузер и перейдите на http://localhost:8000/. Должна открыться страница Django.
  Проверьте админку: http://localhost:8000/admin/.
8. docker compose logs worker # Проверит Celery Worker

```
При каждом пуше в master:
1. Проверяется стиль кода (flake8).
2. Запускаются тесты.
3. Собирается Docker-образ.
4. Выполняется автоматический деплой на сервер.

```

📐 Архитектура базы данных

•  User: Расширенная модель пользователя с gender, birth_date,  age_group,
•  AgeGroup: Справочник возрастных групп.
•  TemporaryUser: Модель временного пользователя для верификации.
•  Event: Модель ананосов в ресторане.
•  Reservation: Модель бронирования столиков
•  TeamMember: Модель-справочник сотрудников в ресторане(с возможностью расширения для рабочих нужд)
•  Table: Модель-справочник столиков в ресторане

📝 Процесс бронирования
 Для зарегистрированных пользователей:
1. Выбор времени и столика через форму.
2. Мониторинг статуса в Личном Кабинете.
3. Email-напоминание за час до визита.
4. После смены статуса на "Завершено" — возможность оставить Score (оценку) и отзыв.

 Для анонимных пользователей:
1. Бронирование через форму или по телефону.
2. Данные сохраняются в системе для менеджера, уведомления приходят на указанную почту.

 Для администрации:
1. Удобная фильтрация броней по статусам и мероприятиям.
2. Возможность оставлять администраторские комментарии (comment_admin) к каждой брони.
Брони анонимного пользователя также сохраняются в системе.

Для админа дополнительно реализовано:
1. Возможность просматривать брони определенных мероприятий

Для запуска тестов
1. docker compose exec app python manage.py test
2. docker compose exec app python manage.py test app
3. docker compose exec app bash -lc "coverage run --source='restaurants' manage.py test restaurants && coverage report -m"
3. docker compose exec app bash -lc "coverage run manage.py test  && coverage report -m"
4. docker compose exec app bash -lc "coverage run manage.py test && coverage report -m && coverage html"
